# zh-jcjq-journal

## 项目简介

本仓库用于归档与发布多个独立的 HTML 页面。这些页面以静态网页形式存储，不针对任何特定域名或具体网站，主要用于记录、展示或分享一些独立的信息内容。

## 目录结构

```
zh-jcjq-journal/
├── index.html          # 项目主页 / 目录页
├── pages/              # 存放各个独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 静态资源文件（CSS、图片等）
    ├── style.css
    └── images/
```

- `index.html` 为仓库的入口页面，可在此处列出所有归档页面链接。
- `pages/` 目录下存放各个独立 HTML 页面，文件名应反映页面内容。
- `assets/` 目录存放页面所需的样式、图片等资源文件。

## 页面归档说明

- 每个 HTML 页面均为独立内容，互不依赖。
- 页面之间可通过超链接相互引用，但不强制要求。
- 归档内容不限于特定主题或格式，以实际提交为准。
- 页面内容应遵守相关法律法规，避免包含敏感或违规信息。

## 维护说明

- 如需新增页面，请在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html` 中的链接。
- 建议在提交时附带简要说明，以便了解页面用途。
- 资源文件请统一存放于 `assets/` 目录，避免分散。
- 本仓库不接受涉及具体域名、商业推广或诱导点击的内容。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务直接访问本仓库中的 HTML 页面。具体访问方式取决于仓库设置及托管平台。
